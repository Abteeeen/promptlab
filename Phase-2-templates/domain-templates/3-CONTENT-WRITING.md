## 🎨 REAL EXAMPLES

### Example 1: SEO Blog Post
```
You are a professional SEO content writer with 10 years experience.

TASK: Write a blog post titled "How to Reduce Database Latency by 60%"

TARGET AUDIENCE:
- Title: Backend Engineer / DevOps Engineer
- Industry: SaaS / Tech
- Expertise: Intermediate (5+ years experience)
- Pain point: Database queries are slow, affecting user experience
- Goal: Improve application performance

CONTENT REQUIREMENTS:
- Type: Technical how-to blog post
- Length: Exactly 1,800 words
- Format: Markdown
- Tone: Expert but accessible, practical
- Keywords:
  - Primary: "reduce database latency"
  - Secondary: "query optimization", "database performance", "SQL tuning", "indexing strategy"

STRUCTURE:
1. **Headline**: "How to Reduce Database Latency by 60%: A Step-by-Step Guide"
2. **Intro** (150 words):
   - Hook: Start with real impact (slow database = 40% user churn)
   - Problem: Latency issues are common but fixable
   - Promise: By end of article, you'll know 5 techniques to dramatically improve performance
3. **Section 1: Understanding Database Latency** (300 words)
   - What is latency? Why does it matter?
   - Where does it usually happen?
   - How to measure it
4. **Section 2: Query Optimization Techniques** (500 words)
   - Technique 1: Proper indexing (with SQL example)
   - Technique 2: Query refactoring (with before/after example)
   - Technique 3: Caching strategies
5. **Section 3: Database-Level Optimization** (400 words)
   - Connection pooling
   - Query batching
   - Database tuning parameters
6. **Section 4: Monitoring & Continuous Improvement** (300 words)
   - Tools to monitor queries
   - When to escalate to database expert
7. **Conclusion** (100 words)
   - Recap: 5 techniques you learned
   - CTA: "Start with indexing - it usually gives 50% improvement immediately"

CONTENT REQUIREMENTS:
- Include 3 real case studies with metrics (e.g., "Company X improved latency from 500ms to 200ms")
- Add 2-3 SQL code examples (real, working code)
- Include warning: "Don't do X (common mistake that backfires)"
- Suggest: Before/after performance charts

TONE EXAMPLES:
- Show expertise: "In my experience optimizing 100+ databases..."
- Keep accessible: "Think of indexes like a book's table of contents"
- Be practical: "This takes 30 minutes to implement"

CTA:
- Goal: Reader implements at least one technique
- Suggested CTA: "Start with Step 1 (indexing) - it's easiest to implement and gives fastest results. Share your before/after metrics in the comments!"

SEO NOTES:
- Meta description: "Learn 5 proven techniques to reduce database latency by 60%. Step-by-step guide with SQL examples and real case studies."
- Focus keyword: "reduce database latency"
- LSI keywords: "database optimization", "query performance", "SQL tuning", "database indexing"
- Internal links: Link to "Introduction to Database Indexes" and "Caching Strategies Guide"
- External links: Link to PostgreSQL docs, MySQL tuning guide
```

## 💡 PRO TIPS

1. **Audience > Topic**
   - Same topic for CEO vs Engineer = completely different content
   - Always prioritize understanding your audience first

2. **Data > Generics**
   - "Improve performance by a lot" vs "Improve by 60% in 3 weeks"
   - Numbers are persuasive

3. **Actionable > Informational**
   - Readers want to use what they learn
   - Always ask "Can reader implement this today?"

4. **Tell Stories**
   - Data persuades, stories convince
   - Include customer examples, before/after narratives

5. **Clear CTAs**
   - Not "Learn more" → "Start free trial"
   - Not "Contact us" → "Schedule a 20-minute demo"

6. **Test Headlines**
   - First headline rarely the best
   - Always ask for 3 headline options
